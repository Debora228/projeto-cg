GL_REPEAT : Ignora a parte inteira da variável e forma um padrão de repetição
GL_ClAMP: A coordenada é suprimida para um valor entre 0.0 e 1.0
GL_CLAMP_TO_BORDER faz a textura parar de ser mapeada além de 0.0 ou 1.0
GL_CLAMP_TO_EDGE utiliza valores texel (pixels de textura) no limite da textura e repete até o limite do polígono
GL_MIRRORED_REPEAT repete a textura para valores além de 0.0 e 1.0
GL_COLOR_BUFFER_BIT: Limpa o buffer de cor
GL_TEXTURE_2D: Indica que o alvo é uma textura 2D
GL_RGBA: Indica um elemento no sistema RGBA
GL_UNSIGNED_BYTE: Indica que o elemento é um byte sem sinal
GL_TEXTURE_MAG_FILTER: Indica que a textura poderá ser ampliada além da área mapeada.
GL_LINEAR: Indica que será retornado a média dos 4 elementos de textura mais próximos.
GL_TEXTURE_MIN_FILTER: É usada sempre que o pixel que está sendo texturado mapeia para uma área maior do que um elemento de textura.  
GL_TEXTURE_WRAP_S: Define o parâmetro de revestimento para as coordenadas da textura
GL_NO_ERROR:  avisa que nenhum erro foi encontrado.
GL_QUADS: Exibe um quadrado preenchido conectando cada quatro pontos definidos por glVertex;
GL_VERTEX_ARRAY: Habilita a matriz(vetor) de vértices para ser escrita e renderizada.
GL_ARRAY_BUFFER: Especifica que o alvo do objeto de buffer é um vetor de vértices.
GL_STATIC_DRAW: Especifica que o conteudo armazenado será modificado uma vez e pode ser usado como fonte para os comandos GL
GL_ELEMENT_ARRAY_BUFFER: Indica que o buffer que você está apresentando, contém os elementos de outro buffer.
